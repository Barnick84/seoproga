Ниже представлен **пошаговый алгоритм автоматической SEO-оптимизации веб-страницы**, адаптированный под реалии поисковых систем 2026 года (с фокусом на Яндекс, но применимый к большинству современных поисковиков). Алгоритм построен на принципах **entity-first индексации, верифицируемого E-E-A-T, AI-качества контента и UX-метрик нового поколения**.

---

### 🔹 Этап 1: Обработка кластера запросов и определение интента

1. **Классификация интента**: информационный, коммерческий, транзакционный, навигационный, локальный.
2. **Сопоставление с воронкой**: определение этапа пути пользователя (awareness → consideration → decision).
3. **Региональная и сезонная корректировка**: привязка к геозапросам (Yandex Wordstat + Yandex Metrica данные), учёт сезонных трендов.
4. **Приоритизация**: баланс объёма, конкуренции, коммерческой ценности и вероятности ранжирования.

---

### 🔹 Этап 2: Анализ ТОП-10 Яндекса (контент + технические + коммерческие сигналы)

1. **Парсинг структуры**: заголовки, блоки, таблицы, списки, медиа, FAQ, отзывы, цены, CTA.
2. **NLP-анлиз покрытия сущностей**: извлечение главных и сопутствующих сущностей, атрибутов, вопросов (`Похожие запросы`, `People Also Ask`).
3. **Технические метрики**: INP, LCP, CLS, мобильная адаптивность, наличие `hreflang`, `canonical`, `robots`, `sitemap`.
4. **Коммерческие и доверительные сигналы**: контакты, политика конфиденциальности, сертификаты, реальные отзывы, авторы/эксперты.
5. **Выявление пробелов**: семантические лакуны, отсутствующие форматы контента, слабые технические параметры у конкурентов.

---

### 🔹 Этап 3: Построение семантического графа и карты сущностей

1. **Ядро страницы**: главная сущность + синонимы + атрибуты.
2. **Связанные сущности**: через API Yandex Knowledge Graph / Wikidata / локальную NLP-модель (ruBERT, Yandex NLP).
3. **Вопросно-ответный слой**: извлечение `H2/H3`-ориентированных вопросов, длинных хвостов, микрозапросов.
4. **Карта интентов**: распределение блоков страницы по этапам пользовательского пути.

---

### 🔹 Этап 4: Генерация и оптимизация контента (AI + E-E-A-T+)

1. **Структурный каркас**: `H1` → вводный абзац (интент) → логические блоки `H2/H3` → FAQ → CTA → источники/автор.
2. **AI-генерация с RAG**: использование локальной или API LLM с привязкой к проверенным источникам, цитатами, оригинальными данными.
3. **Верифицируемый E-E-A-T**:
   - Указание автора с экспертизой, ссылками на профили/публикации.
   - Добавление оригинальных медиа (фото, схемы, расчёты).
   - Чёткое указание источников и даты обновления.
   - При использовании AI: прозрачная пометка или интеграция в workflow с human-in-the-loop валидацией.
4. **Семантическая оптимизация**: естественное вхождение кластера, LSI-вариации, отсутствие переспама, плотность ключей < 2.5%.

---

### 🔹 Этап 5: Техническая и UX-оптимизация (стандарты 2026)

1. **Core Web Vitals 2026**:
   - `INP` < 200 мс, `LCP` < 2.5 сек, `CLS` < 0.1.
   - Новые сигналы: `Interaction Readiness`, `Content Stability`, `Scroll Jank` (отслеживаются через Lighthouse API / Yandex Webmaster).
2. **Мобильность и доступность**: адаптивная вёрстка, WCAG 2.2/3.0, корректные `alt`, `aria-label`, контрастность.
3. **Оптимизация ресурсов**: WebP/AVIF, lazy load, `preload` критических шрифтов, кэширование, CDN.
4. **Яндекс-специфика**: корректные `robots.txt`, `sitemap.xml`, `canonical`, `hreflang`, отсутствие скрытого текста/клоакинга.

---

### 🔹 Этап 6: Разметка и интеграция с Knowledge Graph

1. **JSON-LD 2026**: `Article`, `Product`, `FAQPage`, `HowTo`, `Review`, `Author`, `Organization`, `Breadcrumb`.
2. **Entity disambiguation**: `sameAs`, `@id` с привязкой к Wikidata/Yandex справочнику.
3. **Коммерческая разметка**: `offers`, `price`, `availability`, `aggregateRating` (с валидацией отзывов).
4. **Валидация**: Yandex Rich Results Test / Schema.org Validator / JSON-LD Playground.

---

### 🔹 Этап 7: Валидация на соответствие фильтрам 2026

1. **AI-качество контента**: проверка на `perplexity`, `burstiness`, фактологическую точность, плотность цитирований.
2. **Анти-спам проверки**: отсутствие ключевых повторов, скрытых ссылок, doorway-паттернов, авто-генерации без модерации.
3. **Доверительные сигналы**: контакты, реквизиты, политика, SSL, реальные отзывы, история домена.
4. **Яндекс-специфичные фильтры**: проверка через Yandex Webmaster API (диагностика, рекомендации, статус индексации).

---

### 🔹 Этап 8: Сборка, мета-теги и публикация

1. **Семантический HTML**: `<article>`, `<section>`, `<header>`, `<footer>`, `<nav>`, корректная иерархия заголовков.
2. **Мета-данные**: `title` (до 60 символов, интент + ключ), `description` (до 155, CTA + уникальность), OG/Twitter, `canonical`.
3. **Версионирование и аудит-трейл**: лог изменений, дата обновления, автор, статус проверки.
4. **Отправка на индексацию**: обновление `sitemap`, пинг Yandex Webmaster API, проверка `noindex`/`canonical` конфликтов.

---

### 🔹 Этап 9: Мониторинг и авто-адаптация

1. **Метрики**: позиции, CTR, dwell time, bounce rate, конверсии (Yandex Metrica + Webmaster).
2. **Триггеры обновления**: падение позиций > 15%, появление нового ТОП-контента, смена интента, технические деградации.
3. **A/B тесты**: заголовки, структура блоков, CTA, форматы медиа.
4. **Feedback loop**: обновление кластера, корректировка карты сущностей, переобучение NLP-моделей под актуальные паттерны.

### ⚠️ Критические риски 2026 года

1. **AI-контент без верификации**: Яндекс активно применяет нейросетевые фильтры оригинальности и экспертности. Чистая автогенерация → пессимизация.
2. **Переспам сущностей**: алгоритмы 2026 наказывают искусственное нагромождение ключей/синонимов. Приоритет → естественный контекст.
3. **Игнорирование UX-метрик**: INP и Interaction Readiness напрямую влияют на ранжирование. Медленная страница = потеря трафика, даже с идеальным контентом.
4. **Отсутствие коммерческих сигналов**: для Yandex важны доверие, прозрачность, реальные контакты и отзывы. Без них конверсия и позиции падают.

### ✅ Рекомендации по внедрению

- **Human-in-the-loop**: автоматизируйте 70-80% процесса, но оставьте экспертную валидацию фактов, авторства и коммерческих блоков.
- **Инкрементальные обновления**: не меняйте всю страницу сразу. Обновляйте блоки, отслеживайте реакцию поисковика и пользователей.
- **Аудит-лог**: фиксируйте каждое изменение. Это критично для отладки алгоритмов и соответствия требованиям поисковиков.
- **Тестовый контур**: запускайте оптимизацию на 5-10 страницах, сравнивайте метрики, калибруйте веса в модели перед масштабированием.

# 🔄 ПОШАГОВАЯ РЕАЛИЗАЦИЯ МОДУЛЕЙ

## 1. `intent_classifier.py`

- Вход: поисковый запрос, список URL ТОП-10, опционально SERP API ответ.
- Логика (4 уровня с весами):
  1. Лексика/Regex (15%) – коммерческие/информационные маркеры RU.
  2. SERP-фичи (35%) – наличие виджетов товаров, цен, PAA, Wiki.
  3. Top-10 DOM анализ (35%) – доля e-commerce vs editorial доменов.
  4. LLM-верификатор (15%) – контекстная оценка многозначности.
- Выход: `IntentResult` (intent: commercial|informational|transactional|mixed, confidence: 0.0-1.0, recommendation).
- Кэширование результатов (diskcache/redis).

## 2. `meta_optimizer.py`

- Вход: list competitor_titles, list competitor_descs, target_title, target_desc.
- Логика:
  - Очистка HTML, лемматизация (`pymorphy3`/`natasha`), фильтрация стоп-слов.
  - Анализ частотности в ТОП-10 с порогом ≥30% документов.
  - Gap-анализ: поиск отсутствующих высокочастотных лемм.
  - Учёт лимитов: Title ≤60, Description ≤155 (Яндекс 2026).
  - Генерация предложений с приоритетом по частоте и сохранением читаемости.
- Выход: `MetaOptimizationReport` (current, missing_words, optimized_example, space_left).

## 3. `blueprint_gen.py`

- Вход: list competitor_htmls, keywords, lsi_terms, brand_expert.
- Логика:
  - Парсинг `H2/H3/H4`, привязка текстовых блоков до следующего заголовка.
  - Расчёт медианного объёма текста по ТОП-10.
  - Извлечение 2-грамм и 3-грамм по документной частоте (min_freq=3-4), фильтрация мусора.
  - Объединение синонимичных заголовков через лемматизацию + сохранение иерархии.
  - Добавление обязательных блоков 2026 (Введение, FAQ, Экспертное заключение).
  - Масштабирование `target_words` пропорционально медиане.
- Выход: `ArticleBlueprint` (outline, median_total_words, required_ngrams, keywords, lsi).

## 4. `content_generator.py`

- Вход: `ArticleBlueprint`, config LLM (api_key, model, temperature).
- Логика:
  - Генерация строгого ТЗ для LLM: роли, структура с объёмами, ключи, LSI, обязательные n-grams, анти-спам правила, E-E-A-T требования, формат вывода (чистый HTML).
  - Вызов LLM API (OpenAI/YandexGPT/локальная) с `temperature=0.25-0.3`, `max_tokens=4000`.
  - Fallback: повторный запрос с пониженной temperature при выходе за ±10% объёма.
- Выход: raw HTML текст статьи.

## 5. `tech_auditor.py`

- Вход: URL страницы или локальный HTML, config CWV порогов.
- Логика:
  - Проверка статуса, редиректов (≤2 hops), `robots.txt`, `sitemap.xml`, `canonical`.
  - Запуск Lighthouse (через subprocess) → парсинг INP, LCP, CLS, SEO-score.
  - Валидация JSON-LD структуры (regex + schema.org базовая проверка).
  - Эмуляция User-Agent YandexBot.
- Выход: `TechAuditReport` (cwv_scores, issues: list[Issue(priority, message)], passed: bool).

## 6. `exporter.py`

- Вход: оптимизированный HTML, meta-данные, blueprint, config.
- Логика:
  - Сборка полного HTML (`<!DOCTYPE html>`, `<head>` с meta, OG, canonical, `<script type="application/ld+json">`).
  - Генерация JSON-LD: `Article`, `FAQPage`, `Author`, `Organization` с `sameAs`/`@id`.
  - Экспорт в файл / отправка в CMS API.
  - Ping Yandex Webmaster Recrawl API (OAuth2, user_id, host_id, url).
  - Запись audit-лога: версия, метрики до/после, дата, статус.

# ⚙️ ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ И СТАНДАРТЫ 2026

1. **Типизация**: строгие type hints, Pydantic v2 для всех DTO.
2. **Конфигурация**: `config.yaml` с параметрами порогов, лимитов длины, весов интента, API ключей. Загрузка через `pydantic-settings` или `PyYAML`.
3. **Обработка ошибок**: кастомные исключения (`IntentClassificationError`, `LLMGenerationError`, `AuditFailed`), graceful fallback, логирование с traceback.
4. **Анти-спам и качество контента**:
   - Плотность ключей ≤1.8%, LSI распределены равномерно.
   - Запрет doorway-паттернов (cosine similarity с другими страницами сайта <0.65).
   - Проверка perplexity/burstiness (опционально через локальную LLM).
   - Обязательные E-E-A-T метаданные в контенте и JSON-LD.
5. **Производительность**: `aiohttp` для сетевых запросов, кэширование SERP/n-grams, асинхронный запуск Lighthouse где возможно.
6. **Логирование**: `structlog` или стандартный `logging` с уровнями INFO/ERROR, запись в файл + консоль, audit-trail JSON для каждой страницы.
